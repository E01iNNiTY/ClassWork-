Week 8 Vocabulary

Decomposition
Breaking a large, complex problem into smaller, more manageable parts.
Inheritance (in OOP)
A mechanism where a new class (subclass) inherits properties and behaviors from an existing class (superclass), promoting code reuse and hierarchy.
Algorithmic Thinking
The process of defining a step-by-step solution or algorithm to solve a problem, often breaking it down into clear, logical procedures.
Refactoring
Improving the internal structure of existing code without altering its external behavior, enhancing readability, maintainability, and performance.
Variables
Named storage locations in a program that hold values. These values can change during the execution of the program.
Loops
Control structures that repeat a block of code multiple times until a certain condition is met.
Abstraction
Hiding complex implementation details to expose only the essential features, making systems easier to understand and use.
Relational Operators
Operators that compare two values (like <, >, ==, !=) and return a boolean result (true or false).
Mathematic Operators
Operators used to perform arithmetic calculations, such as + (addition), - (subtraction), * (multiplication), and / (division).
Pattern Recognition
Identifying repeating structures or sequences within data or problems, helping in making predictions and simplifying solutions.
Logical Operators
Operators that combine or invert boolean values, such as AND, OR, and NOT, to form more complex logical expressions.
Conditional Branching Operators
Instructions that direct the program to execute certain parts of code only if specific conditions are met (e.g., if, else, switch statements).
