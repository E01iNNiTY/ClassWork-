Week 13 Vocabulary

Uniform Resource Locator (URL)

A URL is the address used to access resources on the Internet. It specifies the protocol (e.g., http, https), the domain name, and often a path to a specific file or resource.

Firewall

A security system that monitors and controls incoming and outgoing network traffic based on predetermined security rules. Firewalls serve as a barrier between trusted and untrusted networks.

Transmission Control Protocol-Internet Protocol (TCP-IP)

A suite of communication protocols used to interconnect network devices on the Internet. TCP handles reliable data transmission, while IP is responsible for addressing and routing packets.

Network Protocols

Sets of rules and standards that define how data is transmitted and received over a network. They ensure that devices can communicate with each other effectively.

Computer Network

A collection of interconnected computers and devices that communicate with each other to share resources and information.

IPv4

The fourth version of the Internet Protocol, which uses 32-bit addresses to uniquely identify devices on a network. It is one of the core protocols that underpin the Internet.

IPv6

The successor to IPv4, designed to address the shortage of available IPv4 addresses. IPv6 uses 128-bit addresses, allowing for a significantly larger number of unique identifiers.

Domain Name Services (DNS)

A hierarchical system that translates human-readable domain names (like www.example.com) into IP addresses, enabling browsers to load Internet resources.

Hyper-text Transfer Protocol (HTTP/HTTPS)

Protocols used for transmitting hypermedia documents, such as HTML, over the Internet. HTTP is unsecured, while HTTPS provides encryption for secure communication.

File Transfer Protocol (FTP)

A standard network protocol used for transferring files between a client and a server over a computer network.

Software Standards

Agreed-upon guidelines and specifications that software products or processes should adhere to, ensuring consistency, compatibility, and quality across different systems.

OSI Model

The Open Systems Interconnection Model is a conceptual framework used to understand network interactions in seven layers: Physical, Data Link, Network, Transport, Session, Presentation, and Application.
